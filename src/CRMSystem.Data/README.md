# CRMSystem.Data

The **Data Access layer** handles all database operations using Entity Framework Core with SQLite. It depends only on `CRMSystem.Domain`.

## Contents (planned)

- `CrmDbContext.cs` — EF Core DbContext
- `Repositories/`
  - `IRepository.cs` — generic repository interface
  - `Repository.cs` — generic repository implementation
  - `IClientRepository.cs` / `ClientRepository.cs`
  - `IContactRepository.cs` / `ContactRepository.cs`
  - `IUserRepository.cs` / `UserRepository.cs`
- `Configurations/` — EF Core entity configurations (Fluent API)
- `Migrations/` — generated by `dotnet ef migrations add`
